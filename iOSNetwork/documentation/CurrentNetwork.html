benCoding.Network Titanium Module

Using the CurrentNetwork Sub Module

Description

The CurrentNetwork Sub Module provides a wrapper around Apple's the CaptiveNetwork framework.  When you create a new CurrentNework object the module will query the current network interfaces and provide your SSID and BSSID values as readonly properties.

For a detailed example please see the app.js included in the example folder.

Accessing the benCoding.Network Module

Before you can start using any of the module's functionality you need to use the require keyword to make the module accessible from JavaScript. To access this module from JavaScript, you would do the following:

var networkHelpers = require('bencoding.network');

 

Creating an instance of the CurrentNetwork Sub Module

The below example shows how to create the CurrentNetwork object that contains the SSID and BSSID properties

Ti.API.info("Create a Current Network Info Object");
var currentNetwork = networkHelpers.createCurrentNetwork();

 

Object Properties

Property Name Description
SSID Provides access the SSID the device is currently connected.
BSSID Provides access the BSSID the device is currently connected.

 

Example

var networkHelpers = require('bencoding.network');

Ti.API.info("Create a new Current Network Info Object");
var currentNetwork= networkHelpers.createCurrentNetwork();

Ti.API.info("Get SSID => " + currentNetwork.SSID);

Ti.API.info("Get BSSID => " + currentNetwork.BSSID);
